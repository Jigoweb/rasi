Dove vive la configurazione emittente?

Tre modelli architetturali per il pipeline di import. Seleziona quello che si avvicina di più alla tua visione.

A

Config → Transform → Carica

L'admin configura l'emittente una volta (mappatura colonne, tipo episodi, normalizzazioni). Quando si carica un file, il sistema legge la config e trasforma i dati prima che finiscano in programmazioni.

📋 Config DB → 🔄 Transform → 📥 programmazioni → 🎯 Matching
La config decide: quale colonna = titolo, come normalizzare, se la numerazione è continua, ecc.
B

Preview → Valida → Carica

Upload file → sistema mostra anteprima righe con colonne rilevate → admin mappa le colonne interattivamente → sistema trasforma e carica. Config salvata per uso futuro.

📁 Upload file → 👁 Preview → 🗺 Mappa colonne → 📥 Carica
Wizard per ogni upload. Config riutilizzabile ma rivedibile ogni volta.
C

Ibrido: Config salvata + override al volo

L'emittente ha una config salvata (come A). Al momento dell'upload, il sistema mostra un preview delle prime N righe con la config applicata — l'admin può confermare o modificare prima del caricamento definitivo.

📋 Config salvata → 👁 Preview trasformato → ✏️ Override opzionale → 📥 Carica
Veloce per emittenti noti, flessibile per variazioni di formato.
